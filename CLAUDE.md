# CLAUDE.md

## Project Overview

GraphQL Subscriptions API built with Go, [gqlgen](https://github.com/99designs/gqlgen), Echo v5 router, and Redis pub/sub. Includes a JS frontend client.

## Tech Stack

- **Language**: Go 1.26
- **GraphQL**: gqlgen v0.17.86
- **Router**: Echo v5
- **Pub/Sub**: Redis (go-redis/v9)
- **Frontend**: JavaScript (Yarn)
- **Container**: Docker / Docker Compose

## Project Structure

```
server.go              # Entry point
graph/                 # GraphQL schema, resolvers, generated code
internal/
  constants/           # Shared constants (e.g., server port)
  datastore/           # Redis client
  graphql/             # GraphQL server setup
  router/              # Echo router configuration
  service/             # Business logic
frontend/              # JS client application
```

## Build & Development

```bash
make help              # List all targets
make generate          # Generate GraphQL Go source code
make build             # Build the server binary
make run               # Build and run the server
make test              # Run tests (includes generate)
make lint              # Run golangci-lint (includes gocritic) and hadolint
make ci                # Full local CI pipeline: deps, generate, lint, test, build
make ci-run            # Run GitHub Actions workflow locally via act
```

### Infrastructure

```bash
make redis-up          # Start Redis via Docker Compose
make redis-down        # Stop Redis
make image-build       # Build server Docker image
make image-frontend    # Build frontend Docker image
```

### Frontend

```bash
make build-frontend    # Install deps and build JS client
make run-frontend      # Build and start JS client
```

## Code Generation

GraphQL types and resolvers are generated by gqlgen. Configuration is in `gqlgen.yml`. After changing `.graphql` schema files, run `make generate`.

## Linting

- **Go**: golangci-lint v2 with gocritic enabled (configured in `.golangci.yml`)
- **Dockerfiles**: hadolint (configured in `.hadolint.yaml`)

## CI

GitHub Actions workflow (`.github/workflows/ci.yml`) runs on push to `main`, tags, and PRs:

| Job | Triggers | Steps |
|-----|----------|-------|
| **ci** | push, PR, tags | Generate, Lint, Test, Build |
| **docker** | tags only | Build Docker image, Build JS client Docker image |

A separate cleanup workflow (`.github/workflows/cleanup-runs.yml`) removes old workflow runs weekly.
